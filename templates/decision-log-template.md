# {PROJECT_NAME} — Decision Log

**Last updated:** {DATE}

Records every meaningful project decision with context and reasoning. Prevents future sessions from relitigating settled questions. If a decision was made, it goes here — even if it seems obvious now.

---

## How to Use

**When to add an entry:** Any time a choice is made between two or more options, a convention is established, or a direction is set that future sessions need to respect.

**What to capture:** The decision, the alternatives considered, and the reasoning. Future Claude sessions will read this to understand *why* — not just *what*.

**Convention:** Entries are numbered sequentially and never reordered. If a decision is reversed, add a new entry that references the old one (e.g., "Reverses decision #12").

**Note:** For simple projects, the decision log can live inside the roadmap as a "Key Decisions Made" section instead of as a standalone file. Use a standalone file when the project has 20+ decisions and the roadmap is getting long.
