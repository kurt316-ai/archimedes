## How to Use This File

**During a session:** When something goes well, goes wrong, or reveals a reusable pattern — add an entry below. Include enough context that a future Claude session (or Archimedes-builder) can understand the lesson without reading the full conversation.

**Format:** Each entry gets a date, a category, and the lesson. Keep it terse — this is `for-archimedes` style.

**Categories:**
- **Pattern** — Something that worked well and should be repeated
- **Anti-pattern** — Something that went wrong and should be avoided
- **Convention** — A new naming/structure/workflow rule that emerged
- **Tool insight** — Something learned about a specific tool (Notion, Railway, etc.)
- **Workflow** — A process improvement or efficiency gain
- **Glossary term** — A domain term, abbreviation, or concept that emerged in this project and may be useful across projects
- **Archimedes suggestion** — Something Archimedes should add or change
